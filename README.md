2. ¿Cómo se diferencian Flask de Django? Argumentar.
```
Rspta:
- El codigo que usa Flask es más sencillo y utiliza una menor cantidad de líneas de código, esto ayuda a una mejor lectura y compresión del proyecto.
-Django tiene su REST framework que cuenta con un sistema de autenticación de usuarios, mientras que Flask puede usar las opciones nativas para crear un sistema de autenticación básico.
-La velocidad que tiene Flask es superior gracias a su diseño en su estructura.
-Django REST genera APIs que son mejores que la de Flask, pudiendo estos convertirse fácilmente en páginas HTML como puntos finales. Esto significa hacer solicitudes GET y POST fácilmente, algo que en Flask está más condicionado.
```

3. ¿Qué es un API? Explicar en sus propias palabras
```
Rspta:
Una API es una aplicacion que sirve para comunicar y transferir informacion entre otras diferentes aplicaciones en la web. Usa una estructura server-client donde un cliente, por ejemplo, una aplicacion que da la ruta de un pedido que hizo una persona desde su celular sería un cliente, y el servidor es Google maps, donde el cliente consulta los mapas de Google maps y ccon ello dá la ubicacion exacta del pedido. El API es la interface, un codigo de programa, entre ambas aplicaciones donde uno, o sea el cliente, solicita informacion (ubicacion) y el servidor envía esa informacion solicitada.
